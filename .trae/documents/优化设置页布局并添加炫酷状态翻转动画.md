## 1. 优化 options.html 布局与样式
- **全局居中**：修改 `body` 样式，使其使用 `display: flex` 和 `justify-content: center`，在全屏模式下内容居中。
- **Popup 适配**：通过 `body.is-popup` 类恢复弹窗所需的固定宽度（350px）。
- **3D 底部栏**：
    - 创建 `.footer-container` 作为 3D 舞台。
    - 内部包含 `.footer-flip-card`，分为 `.front`（显示“更多设置”）和 `.back`（显示状态文字）。
    - 编写 CSS 动画：当激活 `.showing-status` 类时，执行翻转动作。

## 2. 更新 options.html 结构
- 在 `.container` 底部添加新的 HTML 结构：
```html
<div class="footer-container">
    <div class="footer-flip-card" id="footer-card">
        <div class="footer-side front">
            <a href="#" id="more-settings">更多设置</a>
        </div>
        <div class="footer-side back" id="status-text"></div>
    </div>
</div>
```

## 3. 增强 options.js 逻辑
- **跳转功能**：监听 `more-settings` 的点击事件，跳转至设置页。
- **动画控制**：
    - 重写 `showStatus` 函数。
    - 调用时为 `#footer-card` 添加 `showing-status` 类。
    - 使用 `setTimeout` 在 2 秒后移除该类，使界面翻回“更多设置”。

## 4. 验证与测试
- 在 Popup 模式下测试 3D 翻转动画是否平滑。
- 点击“更多设置”确认是否能正确跳转到全屏设置页。
- 检查全屏设置页的布局是否已正确居中。
